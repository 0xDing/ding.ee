WinFS：一场生不逢时的自我颠覆式创新
------------------------------------------------------

[在 2013 年的一次 Reddit AMA 中](https://www.reddit.com/r/IAmA/comments/18bhme/im_bill_gates_cochair_of_the_bill_melinda_gates/)，比尔·盖茨（Bill Gates）将 WinFS（Windows Future Storage）称为他在微软最大的遗憾。WinFS 曾被视为 Windows Longhorn（即 Windows Vista 的预发布代号）的核心组件之一。微软曾寄望于这一操作系统更新能在桌面计算领域带来根本性的变革。

### 传统文件系统的困境

传统文件系统将数据以字节流的形式封装在文件中。字节如何组织为文件，高度依赖于具体的应用程序。因此，在单个文件内实现数据共享，要求相关应用程序具备统一的解析逻辑。当我们日常使用数以千计、由不同厂商为不同目的开发的应用程序时，这种存储方式便显得力不从心。

以 Windows 资源管理器等文件管理器为例，用户只能通过文件名、扩展名、创建时间等元数据来检索文件，而无法直接根据文件内容进行查找。或许有人会提到 Windows Search 或 Spotlight 之类的方案，但它们的前提是所有支持的文件格式都已集成相应的解析逻辑。而专有文件格式的存在，更是在应用程序之间筑起了一座座数据孤岛。

传统文件系统的另一个典型问题在于：通过树状目录结构来组织数据并不直观。以存放会议记录为例，你需要决定它的归属——是按日期存放，按地点分类，还是以组织者为维度？

尽管 Dropbox 等大多数文件托管服务已经支持文件同步，但其底层仍基于传统文件系统，上述弊端依然存在。遇到同步冲突时，这些服务也只能以整个文件为单位进行覆盖。

### WinFS 的宏大构想

WinFS 是构建于 NTFS 文件系统之上的抽象层，旨在解决上述问题。它本质上是一套带有文件同步能力的统一存储系统，可存储任何数据——无论是结构化、半结构化还是非结构化数据。从某种意义上说，WinFS 是 Dropbox、Zapier 和关系型数据库（RDS）的融合体。

由于应用程序可以在 WinFS 中存储结构化数据，基于内容的搜索便易于实现。即使文件管理器无法识别某种文件格式，仍可通过结构化数据的键值对进行检索。如此一来，应用程序之间的数据共享变得轻而易举。例如，通过定义存储数据的模式（Schema），即时通讯应用便可与电子邮件客户端共享联系人列表。

如同关系型数据库，WinFS 支持数据创建一对一、一对多和多对多的关联关系。这使得定位某份会议记录中提及的所有电子表格成为可能。甚至可以配置实时触发的工作流自动化，向会议记录中提及的人员自动发送邮件。

值得一提的是，WinFS 在设计上也是去中心化的存储方案。其数据可以存储和同步到任何位置——本地、云端或自托管服务器皆可。微软将每台计算机视为个人数据中心，希望以此抗衡谷歌（Google）和脸书（Facebook）等互联网巨头，避免数据存储过度集中化。

与区块链语境中的去中心化不同，WinFS 的去中心化更类似于早期的互联网协议（如 SMTP）。数据仍然存储在集中式的位置上，但你有权选择存储在哪里。无论是存放在云服务还是本地设备上，都不会影响 WinFS 的功能。而且，你可以随时将数据迁移到其他位置。

### WinFS 的命运

遗憾的是，在 Longhorn 屡次延期后，微软决定搁置 WinFS 功能，以确保 Windows Vista 如期发布。这并非微软关于未来的蓝图第一次走向死胡同。早在 [1990 年 11 月 12 日的《纽约时报》](https://www.nytimes.com/1990/11/12/business/the-media-business-microsoft-s-bold-new-game-plan.html)上，微软就曾宣称："在未来，计算机用户将不再关注应用程序本身，而是专注于他们的文档……在这样的设计中，程序将被封装为小型模块，像积木一样自由组合。"其他公司也有过类似的尝试——例如，苹果（Apple）在上世纪九十年代初也曾计划推出 [OpenDoc](https://gregmaletic.wordpress.com/2006/11/12/opendoc/)。

这些尝试大多旨在推动桌面计算从"基于原生应用和非结构化数据"转向"基于复合文档和结构化数据"。然而，这些项目均属自我颠覆式创新，在原生应用仍占据主导地位的年代，它们无一例外地走向了失败。

> "WinFS 的理念超前于它的时代，终将卷土重来。"——比尔·盖茨

复合文档的再度觉醒
-------------------------------------

作为一个较为经典的概念，"复合文档"在不同的历史时期有着不同的定义。简而言之，复合文档是一块可以承载任何内容的画布。其中不仅可以包含富文本，还能嵌入交互式组件、表单或任何其他类型的文档。

从某种意义上说，复合文档与 WinFS 这样的统一存储之间的关系，恰似操作系统中外壳（Shell）与内核（Kernel）的关系。内核为平台提供基础能力，而外壳则使人类得以与内核交互，运用这些能力。

有趣的是，网页本身就是一种典型的复合文档。因此，在 SaaS 席卷一切的时代，在 Web 上重新驱动下一代桌面计算成为可能。人们在网页上花费的时间越来越多。桌面操作系统正受到浏览器的巨大冲击，就连 Figma 和 VSCode 这样的专业应用也纷纷转向浏览器端。

### 互操作性难题

然而，大型企业（员工超过 1000 人）平均同时使用多达 177 种不同的 SaaS 工具。再加上这些 SaaS 内部的数据孤岛，互操作性问题愈加严峻。Zapier 和 Power Automate 等流程自动化工具看似为此而生，但它们更像是在数据孤岛之间往返的渡船，而非真正持久的桥梁。

PWA、WebAssembly、WebGL 等新技术使 Web 能够完成原生应用的大部分功能。Serverless 的流行也推动开发者以更接近复合文档的方式构建 Web 应用。

### 新的构想

软件开发的速度无法跟上用户需求的增长速度，这已成为一场持续性的危机。这一状况催生了面向"公民开发者"（Citizen Developer）的产品热潮，也让"低代码"（Low-Code）和"All-in-One"等概念风靡一时。而在以复合文档为核心的桌面计算中，终端用户可以自由创建复合文档，通过打破软件与文档之间的边界，如搭积木般构建自己的应用。

随着 Web3 的崛起，人们期望 Web 不仅可读、可写，更应可拥有。Web 版的"WinFS"将有助于实现这一目标。万维网之父蒂姆·伯纳斯-李爵士（Sir Tim Berners-Lee）正在推进一个名为 [Solid](https://solidproject.org/) 的新项目来实现这一愿景。IPLD（InterPlanetary Linked Data）作为 IPFS 的子项目，也计划通过弥补 IPFS 在结构化数据方面的不足来达成同样的目标。另一方面，正如鲁本·费尔堡（Ruben Verborgh）在其文章《[通过激励协作实现去中心化](https://ruben.verborgh.org/articles/incentivized-collaboration/)》中所述，Web3 的可编程信任机制也将助力构建可持续的去中心化统一存储。

互联网操作系统：桌面计算的新形态
--------------------------------------

正如用户无法脱离外壳直接使用操作系统内核，复合文档与基于 Web 的统一存储也应被视为一个有机整体。

操作系统是一种系统软件，为应用程序提供通用能力，并确保不同应用之间的互操作性。例如，操作系统提供剪贴板和文件系统来实现应用间的数据共享。因此，广义上的操作系统并不必然与硬件直接相关。

既然复合文档与基于 Web 的统一存储作为整体，能够为构建其上的应用程序提供互操作性和通用能力，那么我们可以将其称为互联网操作系统（Internet OS）。

### 平台迁移已蓄势待发

无论有意与否，Notion、Coda 和 Airtable 等"All-in-One"生产力工具已然成为互联网操作系统的先驱。块编辑器（Block Editor）实际上可以等同于复合文档编辑器。

这些"All-in-One"工作空间的终端用户，最终会感觉它就像一个 BaaS（Backend-as-a-Service），让他们能够创建类似 Dapp 的应用来丰富使用体验。[Microsoft Loop](https://www.microsoft.com/en-us/microsoft-365/blog/2021/11/02/microsoft-office-transforming-for-the-hybrid-world/?culture=en-us&country=US)、[AnyType](https://anytype.io/)、[Hash.ai](http://hash.ai/) 和 [MashCard](https://github.com/mashcard/mashcard) 等近期产品，正如同包裹在统一存储内核外面的外壳。它们能够存储前所未有的丰富数据类型：结构化的数据库块、半结构化的富文本块，以及非结构化的附件块。

你的下一个桌面操作系统，不再是传统意义上的操作系统。蒂姆·伯纳斯-李对 Web 的初始愿景终将实现——Web 将成为安全、去中心化地交换公共与私有数据的介质。而构建于 Web 之上的互联网操作系统，将缔造新时代的 Memex。
